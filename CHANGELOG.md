# Changelog

SparkTrack uses semantic versioning for architectural milestones.

## V1.0

- Established the local-first Windows desktop foundation with Python, PySide6, SQLite, and SQLAlchemy.
- Added the first ontology: Path, Field, Quest, Burst, Artifact, Resource, Season, Principle, and Relationship.
- Introduced dark desktop UI, sidebar navigation, dashboard, and CRUD surfaces.

## V1.5

- Shifted from raw entity management toward operational views.
- Added richer activity, progress, search, command, relationship, and capture services.
- Began separating domain services from UI tables.

## V1.6

- Added momentum, neglect, progress, focus, and context-oriented service concepts.
- Improved the app's ability to summarize lived activity instead of only storing records.
- Preserved local-first storage and future AI extension boundaries.

## V1.7

- Reframed SparkTrack as a Workspace OS.
- Promoted Now, Focus, Quest Cockpit, Field Overview, Season Overview, and Smart View pages.
- Moved database table management toward hidden/developer workflows.
- Added workspace selection, contextual sidebar behavior, quick capture, and command-palette navigation.

## V1.8

- Added Git-ready repository documentation: architecture review, roadmap, changelog, and system context.
- Introduced the Attention Engine as reusable domain infrastructure.
- Added Today Service as the single source of truth for what matters now.
- Formalized Life Feed items as a common activity stream model.
- Refocused Workspace Service as the primary operating-environment abstraction rather than a thin path/field filter.
